GitHub

一，图床

因typora图片一般保存在本地文件，出于可移植性，持久性考虑选择用github作为图床（将图片放在GitHub）

参考链接 PicGo+GitHub使用_我想养只汤姆猫的博客-CSDN博客_picgo怎么用github

操作步骤：

1.GitHub 新建仓库存放图片

2.获取仓库Token

  • 点击生成之后Token只会出现一次，记得保存备用

3.安装PicGo

​ https://github.com/Molunerfinn/PicGo/releases

设定仓库名：你的Github账户名/刚才创建的仓库名

设定分支名：main （guthub之前默认分支名为 master）

设定Token：

指定储存路径：blogImg/ 在仓库中创建一个blogImg的文件夹来存放照片

(因为github包括gitee都不允许创建空文件夹，需在新文件夹下建立一临时文件)

设定自定义域名：https://raw.githubusercontent.com/

自定义域名的作用是，在上传图片后成功后，PicGo会将“自定义域名+上传的图片名”生成的访问链接，放到剪切板上https://raw.githubusercontent.com/用户名/RepositoryName/分支名，，自定义域名可按此填写。

填写完毕确认并设置为默认图库

4.注意事项

  • PicGo上传图片不允许同名文件（StatusCodeError: 422），可在设置中开启时间戳重命名
  • 上传图片失败，无法加载图片
    • 科学上网
    • 修改hosts（github图床链接打开提示raw.githubusercontent.com无法访问解决 - 简书 (jianshu.com)）
      • 1.访问 https://ipaddress.com/ ，在下方搜索框里输入 raw.githubusercontent.com
      • 2.打开 Windows路径下hosts文件(权限) C:\WINDOWS\system32\drivers\etc
      • 3.添加 域名解析 格式如下，注意中间空格 185.199.109.133 raw.githubusercontent.com
      • 4.刷新主机 DNS解析缓存 cmd命令 ipconfig /flushdns
